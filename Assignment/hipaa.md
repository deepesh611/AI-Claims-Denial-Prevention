# HIPAA Overview

## What is HIPAA?

HIPAA (Health Insurance Portability and Accountability Act) is a U.S. law established in 1996 to protect sensitive patient health information.

In simple terms:

> If you handle healthcare data, you must keep it secure and private.

Even outside the U.S., HIPAA is widely used as a benchmark for healthcare data protection, especially in software and AI systems.

---

## What is PHI?

PHI stands for Protected Health Information.

It includes any data that can identify a patient and is related to their health.

### Examples of PHI:
- Name
- Address
- Phone number
- Email
- Medical records
- Lab results
- Insurance details
- Any identifier linked to medical data

---

## Core HIPAA Rules

HIPAA is mainly composed of four important rules:

---

### 1. Privacy Rule

**Focus:** Who can access patient data

This rule defines what qualifies as PHI and regulates how it can be used or shared.

#### Key principles:
- Only authorized individuals can access PHI
- Data should only be used for necessary purposes
- Patients have rights over their own data

---

### 2. Security Rule

**Focus:** How digital health data is protected

This rule applies specifically to electronic PHI (ePHI).

It defines safeguards in three categories:

#### a) Administrative Safeguards
- Policies and procedures
- Access control management
- Employee training

#### b) Physical Safeguards
- Secure devices and servers
- Controlled physical access
- Workstation security

#### c) Technical Safeguards
- Authentication systems (logins)
- Encryption
- Access control
- Audit logs

---

### 3. Breach Notification Rule

**Focus:** What to do if data is exposed

If PHI is leaked or accessed improperly:

- Affected individuals must be notified
- Authorities must be informed
- Action must be taken within a defined timeframe

---

### 4. Enforcement Rule

**Focus:** Penalties for non-compliance

This rule defines consequences for violating HIPAA.

#### Possible penalties:
- Financial fines
- Legal action
- Increased scrutiny and audits

---

## Summary Table

| Rule | Purpose |
|------|--------|
| Privacy Rule | Controls access to PHI |
| Security Rule | Protects electronic PHI |
| Breach Notification Rule | Handles data leaks |
| Enforcement Rule | Defines penalties |

---

## Key Takeaways

- HIPAA is about protecting patient data
- PHI includes any identifiable health-related information
- The Privacy Rule controls access
- The Security Rule ensures protection
- The Breach Rule handles incidents
- The Enforcement Rule ensures accountability

---

## Final Note

You do not need to implement the entire HIPAA law.

Understanding its core principles and applying relevant safeguards is typically sufficient for academic or project demonstrations.
