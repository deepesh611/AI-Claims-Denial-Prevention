### 1.3 Low Level Architecture — Scenario A (Small Load)

> Create a detailed low level AWS architecture diagram for an AI-powered healthcare claim denial prevention system — small hospital variant. Show a dashed arrow from user to Cognito labeled login and a dashed return arrow labeled JWT token. Inside a single AZ AWS VPC show a public subnet with an Application Load Balancer performing SSL termination accepting only HTTPS, connected to an EC2 t3.small running Streamlit dashboard. Show the Streamlit sending REST requests with JWT in the Authorization header to the private subnet. In the private subnet show AWS API Gateway performing JWT signature verification against Cognito and rate limiting, connected to an EC2 t3.medium running FastAPI which extracts the RBAC role from JWT claims and returns 403 on unauthorized access. Show FastAPI calling an AI Agent Orchestrator on the same EC2 which fans out to XGBoost ML model returning denial probability score, and FAISS RAG Retriever returning policy rules. Show AI Agent combining outputs into risk score plus denial reason plus fix steps returned to FastAPI and back to Streamlit. Show FastAPI querying AWS Glue Data Catalog which reads Gold Parquet tables from S3, with S3 showing Bronze Silver and Gold layers and arrows between them labeled Glue ETL job. Show FAISS reading policy embeddings from S3. Show S3 encrypted with SSE-KMS via AWS KMS. Show AWS CloudTrail logging all API and KMS events. Show IAM roles connecting EC2, Glue, and S3 with labels. Use AWS icons.
